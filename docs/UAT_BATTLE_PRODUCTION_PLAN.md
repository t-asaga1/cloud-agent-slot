## 1. 現在のフロー(この形に合わせて絵を作る)

コア(`state.ts` / `scenario.ts`)は無変更。上位 AT のルートは DIRECTION_SPEC 3.6 で実装済み:

- **勝利ルート W1〜W7**(バトル開始時に継続確定): W1・W2 = **義経攻撃** / W3・W4 = **頼朝攻撃** /
  W5〜W7 = **ダブル攻撃**(いずれも「→ 倒せる → 継続」)。
- **敗北寄りルート U1〜U5**(未確定): U1・U2 = **義経攻撃** / U3〜U5 = **頼朝攻撃**
  (いずれも「→ 倒せない → 反撃 → 復活判定」)。**ダブル攻撃に敗北はない = ダブルが出たら勝利確定**
  (下位の「義経強 = 桜花繚乱チャレンジ = 勝利確定」に相当する期待度の頂点)。
- 8G の役割: G1 導入 → G2 義経台詞 → G3 頼朝台詞 → G4 攻撃決め → G5 攻撃 →
  G6 ヒット判定 → G7 帰結 → G8 最終(継続 or 第 3 停止で復活判定)。
- チャンスアップ(G1〜3)はルートへ焼き込み済み(W2 = G2 / W4 = G3 / W6 = G1・G3 / W7 = 全 G、
  U2 = G2 / U4 = G3 / U5 = 全 G)。
- 表示の仕組みは下位と同じ**静止画紙芝居**(レバーオン画像 → 第 3 停止で stop3 画像へ切替。
  `DirectionLayer` の `stoppedReels` 検知)へ移行する。テキスト(技名・台詞・継続/敗北)は
  画像に焼き込まず**アプリ側テキスト描画**(下位の `BattleText` 規約を流用)。

そのため、演出系統(画像の分岐)は次の **5 系統**になる:

| 系統 | ルート | G4〜G8 の流れ |
|---|---|---|
| 義経攻撃・勝ち | W1・W2 | 義経アップ → 蒼炎の技 → 障壁砕け → 後白河崩れる → 継続 |
| 頼朝攻撃・勝ち | W3・W4 | 頼朝アップ → 紫雷の技 → 障壁砕け → 後白河崩れる → 継続 |
| ダブル攻撃(勝利確定) | W5〜W7 | 二人並びアップ → 合体技 → 大爆発 → 後白河吹き飛ぶ → 継続 |
| 義経攻撃・負け寄り | U1・U2 | 義経アップ → 蒼炎の技 → 障壁に防がれる → 赤氷の反撃・被弾 → 敗北(復活判定) |
| 頼朝攻撃・負け寄り | U3〜U5 | 頼朝アップ → 紫雷の技 → 障壁に防がれる → 赤氷の反撃・被弾 → 敗北(復活判定) |

## 2. 画像リスト案(計 25 枚。下位 25 枚と同規模)

下位と同じく「レバーオン画像 → 第 3 停止で stop3 画像」の紙芝居。()内は流用関係。

### G1: 導入 — 2 枚

下位の「雲間の月(青 / 赤)」と統一文法で、**雪原の空に浮かぶ月**にする:

1. `g1_normal` = 吹雪の雪原と夜空の**青い月**。遠景に後白河のシルエット(白い法衣が吹雪に揺れる)。
2. `g1_chance` = 同構図で**赤い月** + 雪原に赤い冷気が漂う(チャンスアップ)。

### G2: 義経台詞 — 1 枚(台詞はアプリ側テキスト。通常 / チャンスで内容差替え)

3. `g2_yoshitsune_serifu` = 雪原に並び立つ義経(手前メイン)と頼朝(斜め後ろ)。義経が刀の柄に手を掛ける。

### G3: 頼朝台詞 — 1 枚(同上)

4. `g3_yoritomo_serifu` = 同ロケーションで頼朝メイン(義経が斜め後ろ)。頼朝が軍配を構え紫の雷が指先に走る。

### G4: 攻撃決め — 4 枚(レバオン = 対峙 → 第 3 停止で攻撃側アップ)

5. `g4_lever_taiji` = **三者の対峙**。手前に義経・頼朝の背中越し、雪原の奥に後白河(赤い氷の結晶が周囲に浮かぶ)。
6. `g4_stop3_yoshitsune_up` = 義経の顔アップ(青い炎が立ち上る)。
7. `g4_stop3_yoritomo_up` = 頼朝の顔アップ(紫の雷をまとう)。
8. `g4_stop3_double_up` = **二人並びの顔アップ**(青炎 + 紫雷が交差。= 勝利確定の合図)。

### G5: 攻撃 — 6 枚(構え → 第 3 停止で技名カット。技名はアプリ側テキストを白帯へ)

9. `g5_yoshitsune_lever` = 義経が蒼炎をまとい斬りかかる構え。
10. `g5_yoshitsune_stop3` = 蒼い炎の斬撃が走る決めカット(白い筆帯スペースを空ける)。技名 =「蒼炎一閃」(仮)。
11. `g5_yoritomo_lever` = 頼朝が軍配を掲げ紫雷を呼ぶ構え。
12. `g5_yoritomo_stop3` = 紫の雷の奔流が奔る決めカット(白帯)。技名 =「紫電轟雷」(仮)。
13. `g5_double_lever` = 二人が同時に構える(青炎と紫雷が渦を巻く)。
14. `g5_double_stop3` = **合体技**の決めカット(青炎 × 紫雷の螺旋。白帯)。技名 =「炎雷共鳴」(仮)。

### G6: ヒット判定 — 4 枚(レバオン = 障壁 → 第 3 停止で成否)

15. `g6_lever_shouheki` = 攻撃が迫り、後白河が**深紅の氷の障壁**を展開して受ける(全系統共通)。
16. `g6_stop3_hit` = 障壁が砕け散り後白河が被弾(勝ちルート)。
17. `g6_stop3_guard` = 障壁に完全に防がれ、後白河が余裕の冷笑(負け寄りルート)。
18. `g6_stop3_double_hit` = ダブル攻撃専用: 障壁ごと大爆発し閃光に包まれる後白河(勝利確定の派手版)。

### G7: 帰結 — 4 枚

19. `g7_win_kuzureru` = 後白河が膝をつき崩れ落ちる(勝ちルート。赤い氷が砕けて舞う)。
20. `g7_double_tobu` = ダブル攻撃専用: 後白河が大きく吹き飛ぶ(雪煙 + 砕けた赤氷)。
21. `g7_lose_hangeki_lever` = **後白河の反撃**。無数の赤い氷柱の槍が二人へ襲い掛かる(負け寄り・レバオン)。
22. `g7_lose_hangeki_stop3` = 義経・頼朝が被弾して雪原に膝をつく(負け寄り・第 3 停止)。

### G8: 最終 — 3 枚

23. `g8_win_keizoku` = 勝どきを上げる義経・頼朝(倒れた後白河が奥に)。「継続」の大文字をアプリ側テキストで重ねる。
24. `g8_lever_down` = 雪原に倒れ伏す二人 + 見下ろす後白河。「敗北」をアプリ側テキストで重ねる(負け寄り・レバオン)。
25. `g8_stop3_fukkatsu_cutin` = **復活カットイン**: 二人が支え合って立ち上がり、青炎と紫雷が再燃する
    (下位の「静カットイン」に相当。全停止後の `revivalCutin` で表示。「復活!」はアプリ側テキスト)。

## 4. 生成方法(承認後の次ラン以降)

- `scripts/gen_battle_images.mjs` へ上位 AT のジョブ(UAT_JOBS)を追加する形で生成
  (`FAL_KEY2` + `openai/gpt-image-2/edit`、quality=high、1792x1008 = 16:9。下位と同一)。
- 参照画像: 設定資料ペア(義経・頼朝)+ **後白河 = `後白河_案A_全身.png` + `後白河_顔アップ.png`**
  (`incoming/reference/キャラクター/黒幕/`)+ 共闘構図(`incoming/reference/背景/その他/共闘構図.png`)。
- 背景参考: **`incoming/reference/背景/上位AT/` は現在空**。候補は
  (a) ユーザーが雪原の参考画像を入稿 / (b) `stage_at_upper.webm` の代表フレームを参照に使う
  (KlingAI 透かし・敵軍が写り込むため「透かし・群衆は描かない」の明示が必要)/
  (c) 参照なしでプロンプトのみ(吹雪の雪原・夜)。**推奨 = (c) + 必要なら (a)**
  (雪原はプロンプトだけで安定して出る画題。ステージ動画との色味合わせは「青白い夜の雪原 + 吹雪」で足りる)。
- 生成 → 原本 25 枚を `incoming/battle/` へコミット(現物確保)→ ユーザー確認 → 組込みラン、の
  下位と同じ 3 段階フロー。

## 7. 生成結果(AGENT #095。2026-07-18)

- Q35〜Q40 承認と同時に、**追加のエンディング・リザルト画像 4 枚**の生成指示あり:
  1. **下位 AT エンディング 1 枚目** = 燃え盛る平等院鳳凰堂が一瞬で赤く凍り付く
  2. **下位 AT エンディング 2 枚目** = 凍り付いた鳳凰堂を砕いて後白河法皇が登場し、義経・頼朝と対峙
  3. **上位 AT エンディング** = 雪原が晴れ渡り、義経と頼朝が笑い合うシーン
  4. **敗北後の共通リザルト** = 義経・頼朝・弁慶・静の全員集合の決めカット
     (実機では**獲得枚数とバトル回数の数値**をアプリ側テキストで重ねる = 画面下部 1/3 を暗めに空けて生成)
- 生成: `scripts/gen_battle_images.mjs` へ **UAT_JOBS(25 枚)+ ENDING_JOBS(4 枚)** を追加
  (`FAL_KEY2` + `openai/gpt-image-2/edit`、quality=high、1792x1008。下位と同一方式)。
  実行 = `node scripts/gen_battle_images.mjs uat` / `... ending`(個別再生成は jobId 指定)。
- キャラデザインは設定資料(義経・頼朝・静・弁慶)+ 後白河 案 A(全身・顔アップ)から
  変えない(プロンプトへデザイン要点と禁止事項を明文化 = `GOSHIRAKAWA_DESIGN` / `BENKEI_DESIGN` を新設)。
- 雪原背景はプロンプトのみで生成(Q38 = (c) 案)。
- **原本 29 枚のコミット先**: 上位バトル 25 枚 = `incoming/battle/上位バトル_*.png` /
  エンディング系 4 枚 = `incoming/ending/*.png`(ジョブ ID ↔ 日本語名 = スクリプトの `JP_NAMES`)。
- 生成の注意: `uat_g7_win_kuzureru` は負傷描写を含む初稿プロンプトが
  content_policy_violation で拒否された(モデレーションは確率的で、同一文言でも
  低解像度では通ることがある)。負傷・苦痛の直接描写を避けた表現へ調整して成功
  (最終プロンプトの正 = スクリプト)。
- **組込みはユーザーの画像確認後の次ラン**(実装方針はセクション 5 のとおり。エンディング系は
  加えて `ending_to_upper` / `ending_complete` 仮ムービーの差し替え + リザルト表示の新設が必要)。
