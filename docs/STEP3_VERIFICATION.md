## 1. 自動チェック(テスト・lint・ビルド)

リポジトリのフォルダでターミナルを開き、以下を順に実行する。

```
npm install        # 初回、または pull 後に package.json が変わっていたら
npm test           # 期待: Tests 274 passed (274)、skip 0(約 60 秒)
npm run lint       # 期待: エラーなし
npm run build      # 期待: エラーなしで dist/ が生成される
```

`npm test` には停止制御の網羅テスト(全役 × 全 20³ 押下位置 × 押し順 6 通り。
対話式停止 = 停止ボタン方式も同粒度で検証)、ステートマシンの単体・通しフローテスト、
固定シード 10 万 G の回帰統計テスト、UI 純ロジック(遊技サイクル / リールアニメ /
メーター / 演出マッピング)のテストが含まれる。これがグリーンならロジックの仕様適合は
機械的に保証される。ブラウザ確認は「見た目・操作・音」の確認が目的。

## 2. アプリの起動

```
npm run dev
```

表示された URL(デフォルト http://localhost:5173)をブラウザで開く。

### 画面構成(STEP 3e の通常プレイ視点レイアウト)

- 中央: 筐体(液晶 = 背景動画 + 演出、リール窓 3 つ)
- 筐体の下: 7seg 風メーターパネル(CREDIT / BET + REPLAY ランプ / WIN / AT 中のみ AT獲得)
- その下: 停止ボタン 3 つ(左 Z / 中 X / 右 C)、レバーオン(Space)・オート消化・BGM ボタン
- ページ下部: 「**デバッグ(開発用)**」の折りたたみ(デフォルト閉)。開くと
  ステータス(モード / 背景 30G カウンタ / フェーズ / 総 G / 差枚 / ナビ + AT 詳細)、
  成立役の強制指定・オート時の押し順・目押しセレクト、ステージ手動切替、リセット、
  発生イベントログ(直近 14 件)、ゲーム履歴(直近 8G)が表示される

### 基本操作

| 操作 | 内容 |
|---|---|
| レバーオン(Space) | BET 3 枚(リプレイ後は自動 BET = REPLAY ランプ)+ 役抽せん + 全リール回転開始 |
| 停止ボタン(Z・X・C) | 押下瞬間に中段にあるコマで停止(タイミング目押し)。押した順 = 押し順 |
| オート消化 | 160ms 間隔でレバー〜全停止を自動実行(適当押し。大量ゲーム確認用) |
| BGM | ステージ連動の BGM 再生 / 停止(切替時はクロスフェード) |

### 3-4. 通しフロー(前兆 → AT → 上位 AT → エンディング → 通常復帰)

短時間で通しを見る手順(実際の確率でも起こるが、強制指定が確実):

1. デバッグを開き、成立役=**中段チェリー(強制)**にして「オート消化 開始」
2. 本前兆(テロップ)→ 連続演出 4G → **AT 突入**(ステージ切替 + AT獲得メーター)
3. 中段チェリー固定なら毎 G V ストックが貯まり、セット継続 → **10 連 → エンディング 10G**(上部バナー)→ **上位AT 突入**
4. 上位 AT 中にもう一度 10 連 → 2 回目のエンディング → **AT終了**(イベントログに
   「AT終了(エンディング完走→…モード・…背景)」)→ 通常背景へ復帰
5. 「オート消化 停止」で止め、イベントログとゲーム履歴で流れを確認する

| 14 | 通常時のステージ | 30G 経過や前兆で背景が自動で移り変わる(デバッグの「背景」に 30G カウンタ表示) |
