# Seedance 予告制作ガイドライン(キャラクター維持ルール)

作成: AGENT #073(2026-07-16)
対象: fal.ai(GPT Image 2 / Seedance 2.0)で予告・演出ムービーを生成するすべての AGENT

fal.ai 接続テスト(AGENT #072)で「設定はほぼ維持されるが**義経の顔だけ変わる**」問題が確認されたため、
本番の予告制作に入る前のルールとして、キャラクターの**顔を含めた同一性の維持方法**を定める。
ByteDance 公式プロンプトガイドおよび 2026-07 時点の Seedance 2.0 運用知見の調査に基づく。

## 1. 結論(必ず守るルール)

1. **キャラクターを出す動画生成では、必ず「設定資料ペア(顔 + 全身)」を参照画像として渡す。**
   - 顔だけのヘッドショット 1 枚 + 全身 1 枚。`incoming/reference/設定資料/` に 4 キャラ分を常備(下記 3.)。
   - 1 枚に複数アングルを並べた「三面図」形式は**使用禁止**(ByteDance 公式が非推奨。モデルが別人の複数人と誤読して顔ドリフトの原因になる)。
2. **プロンプトで @Image タグに役割を明示的にバインドする。**
   - 例: 「@Image1 は義経の顔の参照(顔立ち・髪型はこの画像から一切変えない)。@Image2 は同じキャラクターの全身・衣装の参照。」
   - 参照は**重要な順に前へ**置く(顔 → 全身 → 背景 → スタイルの順。先頭ほど影響が強い)。
3. **参照画像は 4〜5 枚まで**(上限 9 枚だが、枚数を増やすほど 1 枚あたりの注意が薄まり顔が崩れる)。
   - キャラ 2 人なら 顔+全身 × 2 = 4 枚 + 背景 1 枚が上限の目安。3 人以上の同時登場はカット割りで回避を検討。
4. **プロンプトで顔を再描写しない。** 顔の情報は @Image に任せ、テキストは動き・カメラ・シーンの指示に使う
   (テキストの顔描写が参照画像と競合すると顔が変わる = Prompt Leakage)。
5. **激しい頭部回転・高速アクションは顔ドリフトを誘発する。** 顔が重要なカットはミディアム〜クローズアップ +
   穏やかな動きで作り、アクションは別カットに分ける。
6. **image-to-video(1 枚起点)より reference-to-video(@Image バインド)を優先する。**
   AGENT #072 のテストで顔が変わったのは、顔が小さく写ったキービジュアル 1 枚だけを起点にした image-to-video だったことが主因。
7. 生成後は**必ず参考画像と並べて顔を目視検証**(videoReview 併用)し、顔が変わっていたら参照の渡し方・プロンプトを直して再生成する。

## 2. 設定資料画像の仕様(Seedance に最適化した形)

調査で得た「顔維持に最も効く参照画像」の条件(ByteDance 公式 + 運用知見の一致点):

| 項目 | 顔(ヘッドショット) | 全身 |
|---|---|---|
| 構図 | 頭部〜肩のバストアップ・真正面 | 直立ニュートラルポーズ・真正面・頭からつま先まで |
| 表情 | 無表情(ニュートラル)。表情はポーズ焼き付きの原因 | 同左 |
| 背景 | 無地の明るいグレー(透過 PNG は使わない = レンダリング不定) | 同左 |
| 照明 | 影のない均一なスタジオライティング | 同左 |
| 解像度 | 2048×2048(300〜6000px・30MB 以内・アスペクト比 0.4〜2.5 が Seedance の制約) | 1792×2896 |
| 形式 | PNG(JPEG/WebP も可) | PNG |

- 生成は `scripts/gen_character_sheets.mjs`(GPT Image 2 `openai/gpt-image-2/edit`・quality=high)。
  入力はユーザー入稿の参考画像(顔アップ + 正面全身)で、**キャラクターデザインを変えない**指示を固定で含む。
- 再生成: `node scripts/gen_character_sheets.mjs [キャラ名...]` → `/tmp/charsheet/` に出力 →
  参考画像と目視比較してから `incoming/reference/設定資料/` へコピーする(生成のたびに顔の同一性を確認すること)。

## 3. 設定資料の格納場所

`incoming/reference/設定資料/` に 4 キャラ × 2 枚 = 8 枚を常備(AGENT #073 生成・検証済み):

| キャラ | 顔 | 全身 |
|---|---|---|
| 義経 | `義経_顔.png` | `義経_全身.png` |
| 静 | `静_顔.png` | `静_全身.png` |
| 弁慶 | `弁慶_顔.png` | `弁慶_全身.png` |
| 頼朝 | `頼朝_顔.png` | `頼朝_全身.png` |

黒幕は参考画像が未入稿のため未作成(入稿され次第 `gen_character_sheets.mjs` の `CHARACTERS` へ追加して生成する)。

## 4. 動画生成の標準手順(予告 1 本あたり)

1. `fal.storage.upload` で設定資料ペア(+ 必要なら背景参考画像)をアップロード。
2. `bytedance/seedance-2.0/reference-to-video`(検証時は `fast/`)へ `image_urls` = [顔, 全身, (背景)] の順で渡す。
3. プロンプト構成(この順で書く):
   - 参照バインド: 「@Image1 は◯◯の顔の参照(顔立ち・髪型はこの画像から一切変えない)。@Image2 は同じキャラクターの全身・衣装の参照。@Image3 は背景の参照。」
   - シーン・動き・カメラの指示(顔の再描写はしない)。
4. パラメータ: 本番は `resolution` 720p 以上(仮確認は fast + 480p でコスト削減)。`generate_audio` は SE/BGM 別管理の方針(ASSET_GUIDELINES)に従い原則 false。
5. 生成結果の顔を設定資料と並べて検証(videoReview 併用)。NG なら再生成。
6. 採用する動画は従来運用どおり `incoming/` 直下へ入稿 → WebM 変換 → `manifest.json` 登録(参照画像・設定資料はアプリへ取り込まない)。

### プロンプト例(検証済みの雛形)

```
@Image1 は主人公・義経の顔の参照(顔立ち・髪型はこの画像から一切変えない)。
@Image2 は同じキャラクターの全身・衣装の参照。
夜の京都の橋の上、満月を背に義経が静かに立ち、羽織と髪が夜風にゆっくりなびく。
カメラはミディアムクローズアップからゆっくりドリーイン。シネマティックな和風の照明。
```

## 5. 調査メモ(ルールの根拠)

- ByteDance 公式プロンプトガイド: 顔参照は「クリーンなヘッドショット 1 枚 + 全身 1 枚」を推奨。
  多面図・多アングルセットは非推奨(モデルが別人として解釈)。`被写体@image N` のバインド構文が公式仕様。
- Seedance 2.0 の参照上限: 画像 9 枚・動画 3 本(合計 2〜15 秒)・音声 3 本(合計 15 秒以内)、全モダリティ合計 12 ファイル。
  画像は 1 辺 300〜6,000px・30MB 以内・アスペクト比 0.4〜2.5。
- 推奨は最大数ではなく 4〜5 枚(参照が多いほど 1 枚あたりの影響が薄まる)。重要な参照ほどプロンプトの前に置く。
- 顔ドリフトの典型原因: (1) 顔が小さくしか写っていない参照(全身のみ・引きのキービジュアル等)、
  (2) 三面図、(3) プロンプトでの顔の再描写、(4) 激しい頭部運動、(5) 生成物を次の参照に使う連鎖(累積ドリフト)。
  対策はすべて上記 1. のルールに反映済み。
- シーン間の連続性が必要な場合も**毎回同じ設定資料ペアを渡す**(前の生成結果を参照にしない)。
