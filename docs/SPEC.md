## 2. リール仕様

- リール本数: 3
- コマ数: **各リール 20 コマ**(コマ番号 0〜19)
- 表示窓: 縦 3 コマ × 3 リール
- 有効ライン(叩き台): **中段 1 ライン**(近年の AT 機準拠でシンプルに)。チェリーは左リール限定で「角(上段・下段)/中段」の停止形により弱・強を区別する。
- 最大スベリ: 4 コマ(引き込み制御)
- 図柄種類(案・7 種): 赤 7 / 白 7(または機種テーマ図柄)/ BAR / ベル / スイカ / チェリー / リプレイ
  - 実際の図柄はユーザー入稿素材に合わせて確定する。

### 2.1 停止制御(Phase 2 実装済み — `src/core/reel.ts`)

- 方式: **引き込み優先度探索方式**(押下位置から最大 4 コマ先までの 5 候補を探索)。
  優先度は「当選役を揃える > 非当選役ラインを生かさない > スベリが少ない」。非当選役が揃う位置は蹴飛ばす。
- 座標系: 停止位置 = 中段のコマ番号。リールは下方向に回転し、表示窓は 上段 = p+1 / 中段 = p / 下段 = p−1(mod 20)、スベリ k コマで停止位置は 押下位置 + k。
- 停止形の定義:
  - リプレイ / ベル / スイカ: 中段 3 つ揃い
  - **BIG = 赤7 中段揃い / REG = 白7 中段揃い**(取りこぼしあり = 目押し要素)
  - チェリー: 左リール限定。**中段チェリー = 強**、**角(上段・下段)チェリー = 弱**。弱・強とも取りこぼしあり
  - チャンス目: 専用出目は未定義(現状はハズレと同じ「どの役も揃わない」停止)。**確定後に要更新**
- 誤チェリー出目の防止: 当選役がチェリー以外のとき、左リールはチェリーが表示窓に入る位置を原則蹴飛ばす。
  例外として、ライン完成が確定する場合(ベル・リプレイの引き込み、または左最終停止でのライン完成)のみ許容し、表示役判定はライン役を優先する。
  - この制約により、**REG(白7 = 左 12 番、下段に 11 番チェリー)は左リールを最後に押したときのみ揃えられる**。配列変更で解消可能(未調整)。
- 表示役判定(`judgeDisplay`): 中段ライン揃い → 左リールのチェリー形 → ハズレ(`NONE`)の順。取りこぼし時は `NONE`。
- テスト: 当選役 9 種 × 押し順 6 通り × 押下位置 20³ 通りの網羅検証(引き込み可能なら必ず揃う/非当選役は絶対に揃わない/最大スベリ 4 コマ)を `src/core/reel.test.ts` で実施。

## 3. 内部抽選状態(ゲームフロー)

内部抽選状態は全 12 種。**レア役契機とゲーム数(規定 G 数・保証 G 数)契機**で遷移する。

| 区分 | 状態 ID | 名称(仮) | 概要 |
|---|---|---|---|
| 通常 | `NORMAL_A` | 通常 A | 基本状態。CZ/前兆への移行が最も重い |
| 通常 | `NORMAL_B` | 通常 B | CZ 当選期待度が通常 A より高い |
| 通常 | `NORMAL_C` | 通常 C(天国準備等) | CZ/ボーナス直行の期待度が高い |
| CZ | `CZ_LOW` | チャンスゾーン(低) | 規定 G 数消化型。成功でボーナス/AT 抽選 |
| CZ | `CZ_HIGH` | チャンスゾーン(高) | 成功期待度が高い上位 CZ |
| 前兆 | `OMEN` | 前兆 | 本前兆/ガセ前兆。消化後にボーナス告知 or 通常復帰 |
| ボーナス | `BONUS_BIG` | ビッグボーナス | 疑似ボーナス(枚数 or G 数管理)。AT 突入抽選あり |
| ボーナス | `BONUS_REG` | レギュラーボーナス | 小型ボーナス。AT 突入期待度は BIG より低い |
| AT | `AT_MAIN` | AT 本編 | セット管理 or ゲーム数管理(TBD) |
| AT | `AT_UPPER` | 上位 AT | 純増・継続率が強化された AT |
| AT | `AT_EXTEND` | 継続/引き戻しゾーン | セット間の継続バトル・引き戻し区間 |
| AT | `AT_ENDING` | エンディング | 完走時の区間 |

- 状態遷移テーブル(どの状態から・どの契機で・どこへ・何%)は TBD。実装時は `src/core/state.ts` にデータ駆動(JSON/TS テーブル)で持たせ、単体テストで遷移を検証する。
- 遷移契機の種類: レア役(チェリー/スイカ/チャンス目等)、規定ゲーム数到達、CZ/ボーナス/AT の終了、抽選による昇格/転落。

## 4. 演出状態(ステージ)と内部状態の対応

演出(背景・BGM)は内部状態とは**別レイヤー**で管理する。演出ステージは内部状態を示唆するが、1:1 では対応しない。

| 演出ステージ ID | 背景動画 | 主な対応内部状態(示唆) |
|---|---|---|
| `STAGE_NORMAL_A` | 通常背景 A | NORMAL_A/B(B 滞在を弱示唆) |
| `STAGE_NORMAL_B` | 通常背景 B | NORMAL_B/C(高状態を示唆) |
| `STAGE_NORMAL_C` | 通常背景 C | NORMAL_C 濃厚(TBD) |
| `STAGE_CZ_LOW` | CZ 背景(低) | CZ_LOW |
| `STAGE_CZ_HIGH` | CZ 背景(高) | CZ_HIGH |
| `STAGE_OMEN` | 前兆背景(連続演出) | OMEN(ガセ含む) |
| `STAGE_BONUS_BIG` | BIG 中背景 | BONUS_BIG |
| `STAGE_BONUS_REG` | REG 中背景 | BONUS_REG |
| `STAGE_AT_MAIN` | AT 本編背景 | AT_MAIN |
| `STAGE_AT_UPPER` | 上位 AT 背景 | AT_UPPER |
| `STAGE_AT_EXTEND` | 継続バトル背景 | AT_EXTEND |
| `STAGE_AT_ENDING` | エンディング背景 | AT_ENDING |

- 「内部状態 → 演出ステージ」の対応は**確率テーブル(ステージ移行抽選)**で管理し、`src/core/` ではなく演出層(`src/ui/` 配下の presentation ロジック)に置く。ただし抽選自体はテスト可能な純関数として実装する。
- 背景は**ループ動画(WebM)**、ステージごとに 1 本 + BGM 1 曲。演出動画(カットイン等)は別途追加していく。

## 5. 役構成・確率(叩き台 — `src/core/lottery.ts` に実装済み)

- 小役: リプレイ / ベル / スイカ / チェリー(弱・強)/ チャンス目(押し順ベルの有無は TBD。現状は押し順なしの共通ベルとして抽選)
- ボーナス: BIG / REG の直撃抽選あり(CZ・状態経由の抽選は Phase 3 で追加)
- 抽選分母: **65536**(16bit)。テーブルは設定 1〜6 の役別「当選個数」で管理。

| 役 | 設定1 | 設定6 | 設定差 |
|---|---|---|---|

| ベル | 1/6.5 | 1/6.1 | あり(単調) |
| スイカ | 1/60.0 | 1/52.0 | あり(単調) |
| 弱チェリー | 1/75.0 | 1/69.9 | 微差 |
| 強チェリー | 1/300.6 | 1/280.1 | 微差 |
| チャンス目 | 1/199.8 | 1/181.5 | 微差 |

- 設定 2〜5 の数値は `src/core/lottery.ts` の `ROLE_WEIGHTS` 参照(設定 1→6 で単調に良化)。

- **数値を変更する際は `src/core/lottery.ts` と本表を必ずセットで更新すること。**
