## 3. 内部抽選状態(ゲームフロー)

内部抽選状態は全 12 種。**レア役契機とゲーム数(規定 G 数・保証 G 数)契機**で遷移する。

| 区分 | 状態 ID | 名称(仮) | 概要 |
|---|---|---|---|
| 通常 | `NORMAL_A` | 通常 A | 基本状態。CZ/前兆への移行が最も重い |
| 通常 | `NORMAL_B` | 通常 B | CZ 当選期待度が通常 A より高い |
| 通常 | `NORMAL_C` | 通常 C(天国準備等) | CZ/ボーナス直行の期待度が高い |
| CZ | `CZ_LOW` | チャンスゾーン(低) | 規定 G 数消化型。成功でボーナス/AT 抽選 |
| CZ | `CZ_HIGH` | チャンスゾーン(高) | 成功期待度が高い上位 CZ |
| 前兆 | `OMEN` | 前兆 | 本前兆/ガセ前兆。消化後にボーナス告知 or 通常復帰 |
| ボーナス | `BONUS_BIG` | ビッグボーナス | 疑似ボーナス(枚数 or G 数管理)。AT 突入抽選あり |
| ボーナス | `BONUS_REG` | レギュラーボーナス | 小型ボーナス。AT 突入期待度は BIG より低い |
| AT | `AT_MAIN` | AT 本編 | セット管理 or ゲーム数管理(TBD) |
| AT | `AT_UPPER` | 上位 AT | 純増・継続率が強化された AT |
| AT | `AT_EXTEND` | 継続/引き戻しゾーン | セット間の継続バトル・引き戻し区間 |
| AT | `AT_ENDING` | エンディング | 完走時の区間 |

- 状態遷移テーブル(どの状態から・どの契機で・どこへ・何%)は TBD。実装時は `src/core/state.ts` にデータ駆動(JSON/TS テーブル)で持たせ、単体テストで遷移を検証する。
- 遷移契機の種類: レア役(チェリー/スイカ/チャンス目等)、規定ゲーム数到達、CZ/ボーナス/AT の終了、抽選による昇格/転落。

## 4. 演出状態(ステージ)と内部状態の対応

演出(背景・BGM)は内部状態とは**別レイヤー**で管理する。演出ステージは内部状態を示唆するが、1:1 では対応しない。

| 演出ステージ ID | 背景動画 | 主な対応内部状態(示唆) |
|---|---|---|
| `STAGE_NORMAL_A` | 通常背景 A | NORMAL_A/B(B 滞在を弱示唆) |
| `STAGE_NORMAL_B` | 通常背景 B | NORMAL_B/C(高状態を示唆) |
| `STAGE_NORMAL_C` | 通常背景 C | NORMAL_C 濃厚(TBD) |
| `STAGE_CZ_LOW` | CZ 背景(低) | CZ_LOW |
| `STAGE_CZ_HIGH` | CZ 背景(高) | CZ_HIGH |
| `STAGE_OMEN` | 前兆背景(連続演出) | OMEN(ガセ含む) |
| `STAGE_BONUS_BIG` | BIG 中背景 | BONUS_BIG |
| `STAGE_BONUS_REG` | REG 中背景 | BONUS_REG |
| `STAGE_AT_MAIN` | AT 本編背景 | AT_MAIN |
| `STAGE_AT_UPPER` | 上位 AT 背景 | AT_UPPER |
| `STAGE_AT_EXTEND` | 継続バトル背景 | AT_EXTEND |
| `STAGE_AT_ENDING` | エンディング背景 | AT_ENDING |

- 「内部状態 → 演出ステージ」の対応は**確率テーブル(ステージ移行抽選)**で管理し、`src/core/` ではなく演出層(`src/ui/` 配下の presentation ロジック)に置く。ただし抽選自体はテスト可能な純関数として実装する。
- 背景は**ループ動画(WebM)**、ステージごとに 1 本 + BGM 1 曲。演出動画(カットイン等)は別途追加していく。
