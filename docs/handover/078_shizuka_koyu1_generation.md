## 実施内容

### 1. 生成方針の変更(ユーザー指示の反映)

義経版(AGENT #075)は起点静止画で参考背景が平面のバック(書き割り)になっていたため、
静版では**「参考背景の中にキャラクターがいる」奥行きのある合成**へ変更:

- 起点静止画プロンプトで「全身が写るショット + 手前に舞台の床板と欄干 + 背後に本堂・紅葉・
  夕焼け + 足元の床に落ちる自然な影 + 夕焼けの光の色と向きをキャラのライティングに一致 +
  背景画像を平面の書き割りとして貼らず奥行きのある一つのシーンとして合成」を明示。
- 背景参考は `stage_shizuka` の代表フレーム(KlingAI 透かし入り)ではなく、
  入稿参考画像 `incoming/reference/背景/静背景/清水寺_側面.png` を使用。
- 設定資料(`incoming/reference/設定資料/静_顔.png` / `静_全身.png`)+ @Image バインドは
  SEEDANCE_GUIDELINES どおり。

### 2. 生成手順と反復(承認済みプラン「4.」+ 8.1 のとおり)

1. **起点静止画**(GPT Image 2 edit・1792×1008・high): 1 発 OK。顔・衣装は設定資料と一致し、
   清水の舞台の風景の中に立つ構図・足元の影・夕焼けの環境光とも良好。
2. **下書き**(Seedance 2.0 fast 480p・6 秒): 弱 = 1 回で OK。強 = 4 回
   (1 回目: 腕と扇のモーフィング + フラット照明の書き割り感 + 金の光の粒欠落 + 衣装変化 /
   2 回目: 照明・影は解消したが構図が寄りすぎ + 光の粒欠落 / 3 回目: 構図は戻ったが
   扇が袖から湧き出すモーフィング + 光の粒欠落 / 4 回目: 全て解消 = 採用)。
3. **本番**(Seedance 2.0 standard 720p・6 秒): 1 回目は弱・強とも扇と手のモーフィング +
   パン中の 2D スライド(書き割り感)+ 強のエフェクト欠落で不採用。手の役割の明示と
   エフェクトの強調をプロンプトへ追加した 2 回目で弱・強とも採用水準
   (videoReview で確認。下記)。

### 3. 静版で得た生成の教訓(以後の背景・予告にも適用)

1. **手持ち小物(扇)は手の役割まで明示する**: 「最初から胸元に持っている +
   右手でゆっくり開き、左手は体の脇へ下ろす + 袖から新たに出現させない +
   手の指・腕・扇の形は終始崩さない」。曖昧だと手指と小物が溶ける。
2. **強エフェクト(金色の光の粒)は強調 + 英語併記が必要**: 「蛍のように金色に発光する
   無数の小さな光の玉(glowing golden bokeh particles)が最初のフレームから存在し
   漂い続ける」まで書いて初めて描画された(日本語の「金色の光の粒が舞い続ける」だけでは
   3 回連続で欠落)。
3. **環境光・足元の影・衣装維持は動画プロンプト側にも明示する**: 起点静止画が良くても、
   動画側で指定しないと強側でフラット照明・衣装変化が起きた
   (「夕焼けの環境光で自然に照らし、足元の床板に自然な影を落とし、背景と一体の
   奥行きのある画にする。衣装・装飾は @Image2 の参照から一切変えない」)。

最終プロンプトの正は `scripts/gen_yokoku_koyu1.mjs` の `BACKGROUNDS.静`
(教訓は理由コメント付きで記録済み)。

### 4. 採用版の検証結果(videoReview)

- **弱**: 扇を開いて左へ差し伸べ → 左 PAN → 静が画面右へ外れ、欄干と山並みの
  空きスペースで終了。花びら少量。顔・手・背景の破綻なし。
- **強**: 同じ開始構図から右へ差し伸べ → 右 PAN → 空きスペースで終了。
  冒頭から桜吹雪 + 金色の光の粒が最後まで舞い続け、弱との差が明確。破綻なし。
- 両本とも: パン中の視差が立体的で書き割り感なし・開始構図ほぼ一致・顔モーフィングなし。
- **PAN 完了 約 4.0〜4.5 秒・キャラ完全ハケ約 4.5〜5.0 秒** → 現行の
  `KOYAKU_HINT_PRESENTATION` の `symbolDelayMs` = 4.6 秒(KOYU_1)とそのまま整合する見込み
  (組込み後の実機表示で違和感があれば調整)。
