## 1. 方針転換の記録

1. **各予告は動画(一連の動き)ではなく、一旦 3 枚程度の静止画像データにする**。
2. **作る画像の方向性(各予告の内容)はユーザーが指示書で指示を出す**
   (AGENT 側で内容を確定しない。指示書の入稿待ち)。
3. `docs/YOKOKU_PRODUCTION_PLAN.md` へ「12.」として追記し、冒頭に最重要注記を追加。
   「8.」〜「11.」の動画プロンプト案(Q25〜Q30)は**保留**(破棄しない)。
4. 生成済み動画の扱い: 義経背景 固有予告 1(弱・強)= 組込み済みのまま残置 /
   静背景 固有予告 1(弱・強)= 生成済み・確認待ちだったが**組込み保留**
   (動画現物は /tmp のためランと共に消滅済みの可能性大。組込み指示が出た場合は
   `scripts/gen_yokoku_koyu1.mjs` で再生成が必要 = 課金のため必ずユーザーへ確認)。

## 2. 指示書フォーマットの回答(= 今回のユーザーへの回答)

**パワーポイント(pptx)推奨**。

- 理由: (1) 1 スライド = 1 画像にすると説明文と参考画像・ラフスケッチの対応が
  スライド単位で一意になり読み取りの取り違えが起きない / (2) 構図・レイアウト
  (キャラ位置・図柄表示スペースの位置取り)はスライドへ絵や図形で直接描けるため
  画像制作の指示に向く / (3) VM 上で python-pptx により全スライドのテキスト +
  貼付画像(元解像度のまま)を抽出できることを実機検証済み。
- **エクセルでも問題なく読める**(openpyxl でセル値 + 貼付画像 + アンカー位置
  (どの行・列に貼られたか)の抽出を実機検証済み)。予告一覧の表形式管理や
  数値テーブル(振分け値など)を渡す場合はエクセルが確実。
- 推奨スライドテンプレート(8 項目)と入稿方法は `docs/YOKOKU_PRODUCTION_PLAN.md`
  「12.1」に記録: 予告名 + 画像番号 / シーン説明 / 登場キャラ(登場キャラルール準拠)/
  構図・カメラ / 図柄表示スペースの位置 / 弱強差分 / 参考画像・ラフ貼付 / NG 事項。
- 入稿方法: `incoming/specs/` へコミット & プッシュ(従来の仕様書入稿と同じ)。
  画像が多く 25 MiB(ブラウザアップロード上限)を超える場合は GitHub Desktop。

## 3. 実装への影響(未着手。指示書入稿後に設計)

- 現行の予告表示はムービー(webm)前提(`YOKOKU_VIDEOS` + `DirectionLayer`)。
  静止画 3 枚程度への変更で**紙芝居(画像順次切替)方式への UI 変更**が必要になる見込み。
  切替タイミング・レバーオン/停止連動などの仕様は指示書と合わせてユーザーへ確認する。
- 静止画のみなら生成は GPT Image 2(edit)で足りる(Seedance 動画生成は不要)。
  キャラクターを出す場合の設定資料ペア参照・顔検証(SEEDANCE_GUIDELINES)は
  静止画生成でも同様に適用。
- AGENTS.md「AI 素材生成(fal.ai)のルール」(プロンプト事前承認 → 生成 →
  ユーザー確認 → 組込みを別ランで)は静止画生成でも変わらず適用。
