## Step1 ラン ERROR の調査結果(AGENT #012)

- **原因はエージェント実行基盤(Cursor Cloud 側)の異常終了**。最後のツール実行(13:09 UTC、図柄 PNG の寸法確認・正常終了)の後、モデル応答が約 30 分間完了せず、13:39 UTC にラン全体が ERROR 判定された。transcript にエラーメッセージはなく、実装着手前(調査・設計段階)で途切れている。
- ユーザーが見た起動ログの `ERROR Failed to create recording directory /opt/cursor/artifacts/` は VM 起動時の録画ディレクトリに関する**非致命的な警告**で、直後に回復しており無関係(セットアップは `INSTALL_SUCCEEDED`)。
- リポジトリのコード・環境設定に起因する問題ではないため、**リポジトリ側の修正は不要**。Step1 は新しいエージェントで再実行すればよい。
- 誘因として、当該ランはツール実行の合間に極めて長い内部設計(数万文字級の thinking)を繰り返していた。**次回は調査→設計→実装を小さく区切り、こまめにコミット・プッシュすること**(途中でランが落ちても成果が残る)。

## 注意事項

- ブランチは `cursor/<説明>-<suffix>` 形式で作成し、**PR は必ず base を `main` にして作成すること**(suffix は環境から指示される)。
- **PR は 1 本ずつマージすること**(本ファイルが競合しやすい)。競合したら main をブランチへマージして解消する。
- `AGENTS.md` のルール(テスト実行、不要ファイル削除禁止、PR 用の簡潔なまとめ)を守ること。
- **確率・配当・配列などの数値は SPEC.md(Excel + 確定事項)と `src/core/` の実装を必ず一致させる**。抽せんテーブルは分母付き整数(10000 / 100 / 1000)で持ち、行合計 = 分母をテストで検証している。
- **背景移行の 30G は `BACKGROUND_ELAPSED_GAMES`(`src/core/background.ts`)で定数化済み**。値の変更指示が来たら定数と SPEC・テストをセットで更新する。
- 実在機種の素材・画像の流用は禁止(リール配列シートの図柄画像は参考イメージ)。
- ブラウザ API(音声・動画再生・永続化)は `src/platform/` のラッパー経由で使うこと(exe 対応のため)。
- 素材追加時は `docs/ASSET_GUIDELINES.md` に従い `manifest.json` に出所を登録すること。
