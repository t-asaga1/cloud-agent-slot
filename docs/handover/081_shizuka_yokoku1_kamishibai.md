# AGENT #081: 静背景 固有予告 1 を紙芝居(静止画 4 枚)方式で組込み

- 作成日: 2026-07-17
- ユーザー指示: 「静背景専用予告 1、4 つの画像 OK です。組込み方ですが、1 枚目をレバオン、
  2 枚目の弱強は第 1 停止ボタン時に切り替え、第 3 停止ボタン時に 3 枚目 + 小役表示で
  組み込んでみてください」

## 1. 画像の再生成(注意点)

- AGENT #080 の生成 4 枚は**ランのアーティファクトのみ**で、VM 内の現物はラン終了で消滅・
  `incoming/` への入稿もなかった。
- そのため**前ランのトランスクリプト(Cloud Agent 診断 API)から完全同一のプロンプトを
  取り出し、同じ連鎖参照手順(設定資料ペア + 清水寺_側面 → 1 枚目 → 2 枚目弱/強)で
  4 枚を再生成**した(内蔵 GenerateImage = FAL 課金なし)。
- 構図・キャラ・ライティングは承認版と同等だが、**ピクセル単位では承認版と別物**。
- **再発防止**: 採用予定の生成画像は生成ランのうちに `incoming/` へ入稿してもらうか、
  次ラン開始時にチャットへ添付してもらう(HANDOVER 次タスク 6 に記載)。

## 2. 入稿した素材

生成 4 枚(1536x1024 = 3:2)を中央 16:9 クロップ + 1280x720 WebP 化し
`src/assets/images/yokoku/` へ。`manifest.json` 登録済み。

| ファイル(stem) | 用途 |
|---|---|
| `yokoku_shizuka_koyu1_still1` | 弱・強共通 1 枚目(レバーオン)。静・正面チェストアップ |
| `yokoku_shizuka_koyu1_still2_weak` | 弱 2 枚目(第 1 停止)。左向き |
| `yokoku_shizuka_koyu1_still2_strong` | 強 2 枚目(第 1 停止)。バストアップ右向き |
| `yokoku_shizuka_koyu1_still3` | 弱・強共通 3 枚目(第 3 停止 + 小役図柄)。夕焼け空 |

## 3. 実装(紙芝居方式)

- `src/assets/index.ts`: `YOKOKU_IMAGES`(`images/yokoku/*.webp` の glob)を新設。
- `src/ui/direction.ts`:
  - `KoyakuHintView` へ `stills`(leverOn / firstStop / allStop)を追加し `videoUrl` を
    オプショナル化(排他)。
  - 対応表 `KOYAKU_HINT_STILLS`(スロット × 背景 → 静止画 stem。現状 KOYU_1 × SHIZUKA のみ)に
    ある組は紙芝居で解決。**次の予告からはこの表へ 1 行追加 + 画像入稿だけで同じ仕様が乗る**。
- `src/ui/DirectionLayer.tsx`: `stoppedReels` の停止数で切替
  (0 = 1 枚目 / 1〜2 = 2 枚目 / 3 = 3 枚目 + 成立小役の図柄)。
  全画面・図柄即時表示は既存 CSS(`.koyaku-hint-full` / `.hint-symbol`)を流用。
- 図柄はハズレ = ブランク・こぼしベル = ブランク等の既存規約(確定 33 / 39)どおり。
- 詳細仕様と組込み手順は `docs/YOKOKU_PRODUCTION_PLAN.md`「12.4」に記録。

## 4. テスト・確認

- テスト 406 パス(YOKOKU_IMAGES 存在検証 + 紙芝居解決テストを追加)・lint / build グリーン。
- ブラウザ実機確認(デモ用に毎 G 強制表示 → 確認後 revert 済み):
  - 弱(押し順ベル強制): 正面 → 左向き → 夕焼け空 + ベル図柄
  - 強(強スイカ強制): 正面 → バストアップ右向き → 夕焼け空 + スイカ図柄
  - いずれも停止ボタン(Z / X / C)連動で切替、console エラーなし。
