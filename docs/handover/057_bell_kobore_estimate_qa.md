## 仕様変更の整理

| | 現行(SPEC「3.」挙動表・確定どおり) | 変更後(依頼内容) |
|---|---|---|
| 左第一停止 | 必ず上段にベル揃い・**1 枚**(`BELL_PAYOUT_FAIL`) | **12/13**: ベル非揃い(ハズレ目と同様の出目)・**0 枚** / **1/13**: 上段にベル揃い・**13 枚** |
| 中・右第一停止(正解) | 斜め揃い・13 枚 | 変更なし |

- **出玉への影響検証**: 左第一時の期待値は現行 = 1 枚/G、変更後 = (1/13) × 13 = 1 枚/G で**完全に一致**。ユーザーの「機械割・出率に影響なし」の認識は正しい(分散のみ増加 = コイン持ちの荒れが少し大きくなる)。AT 中はナビ遵守(中第一)のため AT 純増 +6.4 も不変。
- 揃った場合の払出が 1 → 13 枚になるため、**表示役ベルの払出は停止形(上段/斜め)に依らず常に 13 枚**となり、`bellSuccess`(1 枚/13 枚の払出区分)は不要になる。

## 実装方針(調査結果)

停止制御エンジン(`reel.ts` の `resolveStop` / `resolveSpin`)は**乱数を持たない純関数**のため、12/13 の抽せんはエンジン内では行わず、上流(レバーオン時)で抽せんしてフラグで渡す。

1. **1/13 抽せんの場所**: レバーオン時にベル当選なら常に乱数 1 個を消費して「左第一時にこぼすか」を決定(押し順に依らず消費 = 決定性の維持。中・右第一なら未使用)。ヘッドレスは `play.ts` の `playGame`、対話式は `App.tsx` のレバーオン(`SpinCycle` にフラグ保持。第一停止まで押し順が未確定のため)。
2. **`reel.ts`**: `BellTarget` を `'TOP' | 'DIAGONAL'` → `'TOP' | 'DIAGONAL' | 'MISS'` へ拡張。`classifyFinal` で MISS のとき「ベル揃い = 禁止出目 / クリーンなハズレ目 = WIN」とする(実装は 1 分岐の追加)。`bellTargetFor` にこぼしフラグ引数を追加し、`resolveStop` / `resolveSpin` のシグネチャへ optional で伝搬。メモキーは bellTarget を既に含むため拡張のみで動く。
3. **蹴飛ばし可能性のリスク = 低**: 「何も揃えず左窓チェリーも出さない」全域蹴飛ばしはハズレの網羅テスト(全 20³ × 押し順 6 通り)で証明済み。MISS はベルだけ蹴ればよい(より緩い条件)ため全域で可能。念のため同粒度の網羅テストを新設して固定する。
4. **`payout.ts`**: 表示役 BELL = 常に 13 枚へ(`BELL_PAYOUT_FAIL` 廃止)。こぼし時は displayed = NONE・0 枚(既存の取りこぼしと同じ経路)。`calcPayout` の `bellSuccess` 引数と `GameState` 入力の `bellSuccess` は削除(または非推奨化)。
5. **出目の見た目**: こぼし時はハズレと同一の蹴飛ばし = DDT 黒バー狙いなら黒バー下段停止などハズレ時と同様の挙動になる(依頼の「外れ目と同様の目」と一致)。
