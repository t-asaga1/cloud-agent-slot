# AGENT #088: 基本背景 固有予告 3 = 会話予告を 4 背景へ組込み完了

- 作成日: 2026-07-17
- ユーザー指示: 「基本背景の背景固有予告 3 の組み込み。4 つの背景全て会話予告。
  一言目は背景キャラ(義経背景なら義経)、二言目は背景キャラ以外の 3 人、
  第 3 停止の全画面は背景キャラ or 頼朝。夕方背景は一言目 = 弁慶 or 義経・
  二言目 = 静・全画面 = 頼朝固定。レバオンで一言目、第 1 停止で二言目、
  第 3 停止で全画面。一言目と二言目は弱の区分で、小役がそろう場合は必ず二言目に行く。
  全画面まで行くパターンが強区分」

## 1. 実装内容(仕様の正 = `docs/YOKOKU_PRODUCTION_PLAN.md` 12.9)

- **素材**: `incoming/yokoku/` の原本 12 枚を WebP 化して
  `src/assets/images/yokoku/yokoku_kaiwa_<char>_<line1|line2|full>.webp` へ
  (char = yoshitsune/yoritomo/shizuka/benkei)。
  - 一言目・二言目(ウィンドウ用)= 3:2 のまま 1152x768。
  - 全画面 = 12.8 の推奨どおり**上端基準 16:9 クロップ(上 0 / 下 160px)**+ 1280x720
    (義経の髪・頼朝の冠が切れないことを確認済み)。
  - `manifest.json` へ 12 件登録。**組込み完了により原本 12 枚は削除**(12.4 の運用)。
    枠 `会話予告_枠.png` は `gen_yokoku_kaiwa.mjs` の再生成参照用に残置。
- **キャスト抽せん**: `drawKaiwaCast(rng, background)`(`src/ui/direction.ts`)。
  義経/静/弁慶背景 = 一言目 背景キャラ固定 / 二言目 他 3 人から均等抽せん /
  全画面 背景キャラ or 頼朝(1/2)。夕方背景 = 一言目 弁慶 or 義経(1/2)/
  二言目 静固定 / 全画面 頼朝固定。`App.tsx` が `drawKoyakuHint` の結果が
  KOYU_3 のとき演出専用 rng(hintRng)で呼び、`koyakuHintView` へ渡す。
- **強弱の解決**(`koyakuHintView` の KOYU_3 分岐 → `KoyakuHintView.kaiwa`):
  - 弱 + 小役がそろう役 → 一言目 + 二言目。弱 + そろわない役 → 一言目のみ。
  - **そろう役の実装解釈 = リプレイ / ベル(こぼし除く)/ スイカ / チェリー**
    (`KAIWA_ALIGNED_ROLES`)。ハズレ・ベルこぼし(bellMiss)・チャンス目
    (テンパイはずし)・リーチ目は「そろわない」扱い。
  - 強 → 一言目 → 二言目 → 全画面(そろわないレア役でも全段階進む)。
  - **弱強の見た目差は第 3 停止まで出ない**(レバオン時点で悟らせない)。
    キャストなしで KOYU_3 を解決しようとすると即エラー(呼び忘れ検知)。
- **表示**(`DirectionLayer.tsx` + `App.css` の `.kaiwa-yokoku` 系):
  レバオン = 一言目ウィンドウ(液晶左上・52% 幅)→ 第 1 停止 = 二言目ウィンドウ
  (右下)を一言目を**消さずに**追加(同時表示・重なり許容 = 12.8)→ 第 3 停止 =
  全画面(強のみ)+ 大台詞(左上)+ 成立役の図柄(右下・即時表示)。
  切替は既存の `stoppedReels` prop(紙芝居方式と同じ配線)。
- **台詞・話者名はアプリ側テキスト描画**(画像に焼き込まない = 12.7 の方針):
  台詞テーブル = `KAIWA_LINES`(direction.ts。**仮セリフ**)。話者名は枠画像
  左上のネームプレートへ重ねる。フォントはウィンドウ幅基準(cqw)。
  **正式台詞への差し替えは `KAIWA_LINES` の変更だけでよい**。
- 不使用化: `yokoku_<bg>_koyu3_weak/strong` ムービー仮素材 8 本は KOYU_3 が
  会話予告になったため不使用(**ファイルは残置。削除はユーザー確認後** = 12.7)。
