# 引継ぎ資料(履歴 #008: PR #7 / #8 の競合解消)

- 作成者: AGENT #008(PR 競合解消)
- 作成日: 2026-07-09

## 発生した問題

- PR #7(筐体取り込み + リール制御 + 筐体レイアウト試作)と PR #8(素材取り込み + 仮素材生成 + リール制御)がオープン中に、PR #9(Excel 仕様書の入稿方法整備)が先にマージされた。
- PR #9 も `docs/HANDOVER.md` を更新していたため、PR #7 / #8 の両方が `docs/HANDOVER.md` で競合し、マージ不能になった。

## 実施した対応

- PR #7 / #8 の各ブランチに `origin/main` をマージし、`docs/HANDOVER.md` の競合を解消:
  - PR #9 の成果(AGENT #006 = Excel 入稿整備)を「これまでの経緯」に統合。
  - 各ブランチの作業者を AGENT #007 に改番し、履歴ファイルも `006_*.md` → `007_*.md` にリネーム。
  - Excel 仕様書入稿時のタスク(openpyxl での読み取り → SPEC 反映)を「次の AGENT へのタスク」に引き継ぎ。
- 両ブランチで `npm test` / `npm run lint` / `npm run build` が全てパスすることを確認してプッシュ済み。

## 重要: PR #7 と PR #8 はどちらか一方のみマージすること

両 PR は**同じタスクを別アプローチで実装した並行 PR** であり、`src/core/reel.ts`・`src/App.tsx`・`src/assets/` など 10 ファイル以上で互いに競合する。

- 片方をマージしたら、もう片方は**マージせずクローズする**こと(マージすると再び競合し、実装も二重になる)。
- 概要比較:
  - **PR #7**: 筐体画像を透過抜き加工 + はめ込み座標 JSON、`CabinetPrototype` UI、リール網羅テスト 22 件(計 46 件)。既知の制約: REG(白7)は左リール最終停止時のみ揃う。
  - **PR #8**: 筐体取り込みに加えて**仮素材一式 39 件**(図柄・ステージ動画・BGM/SE)を生成、`src/platform/audio.ts`(音声ラッパー)実装、リール網羅テスト 18 件(計 42 件)。

## 教訓(再発防止)

- `docs/HANDOVER.md` は全 AGENT が更新するため、**PR を複数オープンにしたまま順不同でマージすると必ず競合する**。PR は 1 本ずつマージし、次の AGENT を起動する運用を推奨。
- 同じタスクを複数 AGENT に並行依頼すると成果物が競合する。並行させる場合はタスクを分割すること。
