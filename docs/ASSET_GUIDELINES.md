# 素材管理ガイドライン(筐体画像・音声・液晶データ)

作成: AGENT #002
対象: 本プロジェクトで使用するすべてのバイナリ素材(画像・音声・動画・フォント等)

## 1. 基本方針

1. **素材はすべてリポジトリ内で一元管理する**(外部ストレージや URL 直リンクは使わない)。
   - Web 版・exe 版の両方でビルド時にバンドルされるため、ビルドに必要なものがリポジトリに揃っていることが引継ぎ開発の前提になる。
2. **オリジナル素材のみを使用する**。実在パチスロ機種の筐体写真・ロゴ・サウンド・映像の流用は著作権・商標の観点から禁止。
3. **すべての素材の出所をマニフェストに記録する**(誰が・何で作ったか、ライセンスは何か)。
4. **「コードで描けるものはコードで描く」を優先する**。バイナリ素材はリポジトリを重くし差分レビューもできないため、最終手段とする。

## 2. 素材の種類ごとの方針

| 素材 | 第一候補 | 理由 |
|---|---|---|
| 筐体(フレーム、ボタン、レバー) | SVG / CSS で描画 | 拡大縮小に強くテキスト差分で管理できる。exe/Web 両対応 |
| リール図柄 | SVG(1 図柄 1 ファイル) | 差し替え・追加が容易 |
| 液晶演出 | まずは DOM/Canvas + CSS アニメーション。必要になったらスプライトシート(WebP)+ JSON 定義 | 演出データを「画像+タイムライン定義(JSON)」に分離するとロジックからテスト可能になる |
| 効果音・BGM | Web Audio API による合成(コード生成)を優先。難しい音のみ音声ファイル(OGG/MP3) | 合成ならバイナリ不要で差分管理できる |
| フォント | システムフォント + Google Fonts(SIL OFL のもの)をローカル同梱 | exe 版はオフライン動作するため CDN 参照は不可 |

## 3. ディレクトリ構成と命名

```
src/assets/
  images/
    cabinet/     # 筐体パーツ(frame.svg, lever.svg, button_stop_left.svg ...)
    reels/       # リール図柄(symbol_seven_red.svg, symbol_bell.svg ...)
    lcd/         # 液晶演出用スプライト・背景(lcd_bg_normal.webp ...)
  audio/
    se/          # 効果音(se_lever_on.ogg, se_payout.ogg ...)
    bgm/         # BGM(bgm_normal.ogg, bgm_bigbonus.ogg ...)
  fonts/         # 同梱フォント
  lcd-data/      # 液晶演出のタイムライン定義(JSON。バイナリではないがここに集約)
  manifest.json  # 全素材の出所・ライセンス台帳
```

- ファイル名は `snake_case`、英語、内容がわかる名前(`se_lever_on.ogg` など)。
- 参照はハードコードせず、`src/assets/index.ts` に定義した ID 経由で行う(Vite の import で解決させ、ビルド時に存在チェックが効くようにする)。

## 4. フォーマットとサイズ上限

| 種別 | フォーマット | 上限(1 ファイル) |
|---|---|---|
| ベクター画像 | SVG | 100 KB |
| ラスター画像 | WebP(透過が必要なものも WebP) | 500 KB |
| 音声 | OGG Vorbis(互換用に必要なら MP3 併用) | SE: 200 KB / BGM: 2 MB |
| フォント | WOFF2 | 1 MB |

- リポジトリ全体のバイナリ合計は **50 MB を目安**とする(exe 配布サイズと clone 時間に直結するため)。
- 上限を超える素材が必要になった場合は Git LFS の導入を検討し、`docs/HANDOVER.md` に判断を記録する。**現時点では LFS は導入しない**(上記方針でバイナリを最小化できる想定のため)。

## 5. 出所管理(manifest.json)

素材を追加・変更したら必ず `src/assets/manifest.json` に 1 エントリ追加する:

```json
{
  "id": "se_lever_on",
  "path": "audio/se/se_lever_on.ogg",
  "type": "audio",
  "origin": "generated",
  "tool": "コード合成(scripts/gen_audio.ts)/ AI 生成(モデル名)/ 手作り",
  "license": "project-original",
  "note": "レバーオン時の効果音"
}
```

- `origin` は `generated`(コード/AI 生成)、`original`(人が制作)、`third-party`(外部素材)のいずれか。
- `third-party` の場合はライセンス名と入手元 URL を必須とし、**再配布可能なライセンス(CC0、SIL OFL 等)のみ許可**。exe として再配布するため、非商用限定・再配布禁止の素材は使えない。

## 6. AGENT 向け運用ルール

- 素材を追加する PR では、スクリーンショット等で見た目/音の確認結果を添付する。
- 既存素材の削除・差し替えは、参照箇所(`src/assets/index.ts` と使用コンポーネント)をすべて更新し、`npm run build` が通ることを確認してから行う。
- 素材生成に使ったスクリプト(音声合成、スプライト生成等)は `scripts/` に残し、再生成可能な状態を保つ。
