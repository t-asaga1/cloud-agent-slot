# 素材管理ガイドライン(筐体画像・音声・動画・液晶データ)

作成: AGENT #002 / 更新: AGENT #003(ユーザー実素材の入稿対応)
対象: 本プロジェクトで使用するすべてのバイナリ素材(画像・音声・動画・フォント等)

## 1. 基本方針

1. **素材はすべてリポジトリ内で一元管理する**(外部ストレージや URL 直リンクは使わない)。
   - Web 版・exe 版の両方でビルド時にバンドルされるため、ビルドに必要なものがリポジトリに揃っていることが引継ぎ開発の前提になる。
2. **オリジナル素材のみを使用する**。実在パチスロ機種の筐体写真・ロゴ・サウンド・映像の流用は著作権・商標の観点から禁止。
3. **すべての素材の出所をマニフェストに記録する**(誰が・何で作ったか、ライセンスは何か)。
4. **ユーザー提供のオリジナル素材を第一とする**。ユーザーから素材(筐体画像・背景動画・図柄・BGM・SE)が提供される前提となったため、コード描画(SVG/CSS/Web Audio 合成)は「素材が未提供・不足している箇所の暫定表示」および「単純な UI パーツ」に限定して使う。
5. 入稿された素材はそのまま使わず、**必ず後述のフォーマット・サイズ規定に変換(トランスコード)してから `src/assets/` に取り込む**。変換は AGENT が ffmpeg 等で行う。

## 2. 素材の種類ごとの方針

| 素材 | 管理方法 | 備考 |
|---|---|---|
| 筐体(フレーム、ボタン、レバー) | ユーザー提供画像(WebP/PNG→WebP 変換)。パーツ分割が必要なら AGENT が加工 | ボタン等の可動部は別レイヤー画像に分けると実装しやすい |
| リール図柄 | ユーザー提供画像(1 図柄 1 ファイル、透過 WebP) | SVG 提供ならそのまま。ラスターなら透過 WebP に統一 |
| 背景動画(状態=ステージごと) | WebM(VP9)。ループ再生前提、**音声トラックなし**(音は BGM で別管理) | 演出ステージ 1 つにつき 1 ファイル |
| 演出動画(カットイン等) | WebM(VP9)。原則音声なしで SE を別ファイル分離(同期が難しいものは音声込み可) | 再生タイミングは JSON 定義(`lcd-data/`)で制御 |
| 効果音・BGM | ユーザー提供音源を OGG Vorbis に変換 | BGM はループポイントが必要なら秒数を manifest に記録 |
| フォント | システムフォント + Google Fonts(SIL OFL のもの)をローカル同梱 | exe 版はオフライン動作するため CDN 参照は不可 |

## 3. 素材のアップロード(入稿)方法

ユーザーが素材を提供する手順。**リポジトリの `incoming/` ディレクトリに素材を置いてコミットする方式**を正とする。

### 手順

1. GitHub 上で(またはローカルで clone して)`incoming/` ディレクトリに素材ファイルを追加し、`main` あてのブランチ/PR としてプッシュする。
   - GitHub の Web UI なら: リポジトリページ → `Add file` → `Upload files` でドラッグ&ドロップ → ブランチを切って PR 作成、が最も簡単。
   - まとめて zip で置いても構わない(AGENT が展開する)。
2. `incoming/` 直下は自由な構成でよいが、**内容がわかるフォルダ/ファイル名**にする(例: `incoming/cabinet/筐体.png`, `incoming/bgm/通常A.wav`)。日本語ファイル名のままでよい。
3. 各素材が「どの状態・どの用途向けか」を `incoming/README.md`(またはコミットメッセージ / AGENT への指示文)に書き添える。
   - 例: 「`bg_normal_a.mp4` は通常ステージ A のループ背景」「`cutin_01.mp4` は激アツカットイン」
4. 次の AGENT に「incoming の素材を取り込んで」と指示する。AGENT が以下を行う:
   - フォーマット変換(下記 4. の規定へ)・リネーム(`snake_case` 英語)
   - `src/assets/` への配置と `manifest.json` への登録
   - 取り込み完了後、`incoming/` 内の元ファイルは削除する(履歴には残る)

### 入稿時の推奨フォーマット(変換前の元素材)

| 素材 | 推奨 | 備考 |
|---|---|---|
| 筐体画像 | PNG(可能なら透過・パーツ分割済み) | 1 枚絵でも可。分割は AGENT 側で対応 |
| リール図柄 | PNG 透過 or SVG、1 図柄 1 ファイル、正方形推奨(例 400x400 以上) | 20 コマ分の配列順はテキストで別途指定 |
| 背景動画・演出動画 | MP4(H.264)or MOV、1280x720 程度、背景はループ前提(先頭と末尾が繋がる) | リポジトリ取り込み時に WebM に変換・圧縮する |
| BGM | WAV / MP3(ループ想定の場合はループ位置をメモ) | OGG に変換して取り込む |
| SE | WAV / MP3 | 同上 |

### サイズが大きい場合

- 動画を含むため、**素材 PR の合計が 100 MB を超えそうな場合は複数 PR に分割**する(GitHub は 1 ファイル 100 MB 制限、警告 50 MB)。
- 1 ファイルで 100 MB を超える素材は解像度・ビットレートを下げてから入稿する。それでも困難なら Git LFS 導入を検討し、`docs/HANDOVER.md` に判断を記録する。

## 4. ディレクトリ構成と命名

```
incoming/        # ユーザー入稿用の一時置き場(AGENT が取り込み後に削除)
src/assets/
  images/
    cabinet/     # 筐体パーツ(cabinet_frame.webp, button_stop_left.webp ...)
    reels/       # リール図柄(symbol_seven_red.webp, symbol_bell.webp ...)
    lcd/         # 液晶用静止画(lcd_bg_fallback.webp ...)
  video/
    stage/       # 状態別ループ背景動画(stage_normal_a.webm ...)
    effect/      # 演出動画(effect_cutin_strong.webm ...)
  audio/
    se/          # 効果音(se_lever_on.ogg, se_payout.ogg ...)
    bgm/         # BGM(bgm_normal_a.ogg, bgm_at_main.ogg ...)
  fonts/         # 同梱フォント
  lcd-data/      # 液晶演出のタイムライン定義(JSON。バイナリではないがここに集約)
  manifest.json  # 全素材の出所・ライセンス台帳
```

- ファイル名は `snake_case`、英語、内容がわかる名前(`se_lever_on.ogg` など)。
- 参照はハードコードせず、`src/assets/index.ts` に定義した ID 経由で行う(Vite の import で解決させ、ビルド時に存在チェックが効くようにする)。

## 5. フォーマットとサイズ上限(リポジトリ取り込み後)

| 種別 | フォーマット | 上限(1 ファイル) |
|---|---|---|
| ベクター画像 | SVG | 100 KB |
| ラスター画像 | WebP(透過が必要なものも WebP) | 500 KB |
| 動画 | WebM(VP9、背景・演出とも原則音声なし) | 背景ループ: 8 MB / 演出: 15 MB |
| 音声 | OGG Vorbis(互換用に必要なら MP3 併用) | SE: 200 KB / BGM: 2 MB |
| フォント | WOFF2 | 1 MB |

- 動画の目安: 1280x720・30fps・VP9。背景ループは 10〜20 秒、ビットレート 1〜2 Mbps 程度。
- リポジトリ全体のバイナリ合計は、動画素材を含むため **300 MB を目安**に引き上げる(従来 50 MB)。exe 配布サイズと clone 時間に直結するため、超えそうな場合は解像度/ビットレートの再圧縮を先に検討する。
- それでも収まらない場合は Git LFS の導入を検討し、`docs/HANDOVER.md` に判断を記録する。**現時点では LFS は導入しない**。

## 6. 出所管理(manifest.json)

素材を追加・変更したら必ず `src/assets/manifest.json` に 1 エントリ追加する:

```json
{
  "id": "se_lever_on",
  "path": "audio/se/se_lever_on.ogg",
  "type": "audio",
  "origin": "generated",
  "tool": "コード合成(scripts/gen_audio.ts)/ AI 生成(モデル名)/ 手作り",
  "license": "project-original",
  "note": "レバーオン時の効果音"
}
```

- `origin` は `user-provided`(ユーザー入稿)、`generated`(コード/AI 生成)、`original`(人が制作)、`third-party`(外部素材)のいずれか。ユーザー入稿素材は `user-provided` + `license: "project-original"` とし、`note` に入稿時の元ファイル名と用途を記録する。
- `third-party` の場合はライセンス名と入手元 URL を必須とし、**再配布可能なライセンス(CC0、SIL OFL 等)のみ許可**。exe として再配布するため、非商用限定・再配布禁止の素材は使えない。

## 7. AGENT 向け運用ルール

- 素材を追加する PR では、スクリーンショット等で見た目/音の確認結果を添付する。
- 既存素材の削除・差し替えは、参照箇所(`src/assets/index.ts` と使用コンポーネント)をすべて更新し、`npm run build` が通ることを確認してから行う。
- 素材生成・変換に使ったスクリプト/コマンド(音声合成、ffmpeg 変換等)は `scripts/` に残し、再生成可能な状態を保つ。
- `incoming/` に未取り込み素材が残っている場合は、最優先で取り込み作業を行う。
