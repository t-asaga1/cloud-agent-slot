# STEP 5 完了時点の動作確認手順(ユーザー向け・Windows 実機確認)

作成: AGENT #053(2026-07-14)
対象: STEP 5(exe パッケージング + CI ビルド = 5a〜5b)完了時点の main ブランチ。
所要時間の目安: exe の入手 5 分 + 起動・チェックリスト 10〜15 分。

**この手順書の目的**: Windows デスクトップ版(Tauri)の実機確認は開発 VM(Linux)では
不可能なため、ユーザーの Windows PC での確認を依頼するもの。ゲーム内容の詳細確認は
`docs/STEP3_VERIFICATION.md` のチェックリストがそのまま使える(本書は exe 特有の確認に絞る)。

## 0. 前提(環境)

- Windows 10 / 11 の PC(64bit)
- WebView2 ランタイム(Windows 11 と最近の Windows 10 には標準搭載。NSIS インストーラは
  無ければ自動でダウンロード・導入する)
- ビルドは GitHub Actions が行うため、**手元に Node.js や Rust は不要**

## 1. exe の入手方法

CI(GitHub Actions の `windows-build` ワークフロー)が 3 種類の成果物を生成する:

| ファイル | 種類 | 用途 |
|---|---|---|
| `yoshitsune-monogatari_0.1.0_x64-setup.exe` | NSIS インストーラ | 通常のインストール(推奨) |
| `yoshitsune-monogatari_0.1.0_x64_en-US.msi` | MSI インストーラ | NSIS で問題がある場合の代替 |
| `yoshitsune-monogatari_portable.exe` | ポータブル exe | インストール不要。そのまま実行 |

### 方法 A: Release から(タグ push 時。配布用)

1. リポジトリで `git tag v0.1.0 && git push origin v0.1.0` のようにタグを push する
   (GitHub の Releases 画面 → Draft a new release → タグ作成でもよい)
2. Actions の `windows-build` が完走すると、Releases ページに該当タグの Release が作成され
   上記 3 ファイルが添付される
3. Release ページからダウンロード

### 方法 B: Actions の成果物から(手動実行。検証用)

1. GitHub のリポジトリ → **Actions** タブ → 左の **windows-build** を選択
2. **Run workflow** ボタン(ブランチ = main)で手動実行
3. 完走したランを開き、ページ下部 **Artifacts** の `yoshitsune-monogatari-windows` を
   ダウンロード(zip。展開すると 3 ファイルが入っている)

## 2. 起動手順

- **ポータブル exe**: `yoshitsune-monogatari_portable.exe` をダブルクリックするだけ
- **インストーラ**: `..._x64-setup.exe` を実行 → 画面に従ってインストール →
  スタートメニューの「yoshitsune-monogatari」を起動

初回起動時に「WindowsによってPCが保護されました」(SmartScreen)が出る場合がある
(コード署名なしの自動ビルドのため正常)。「詳細情報」→「実行」で起動する。

## 3. Windows 実機チェックリスト

VM(Linux)では確認できなかった **#1・#2 が最重要**(開発 VM 特有の制約との切り分け)。

| # | 手順 | 期待する結果 |
|---|---|---|
| 1 | デバッグを開き成立役=中段チェリー(強制)→「1G消化(オート)」連打で AT 突入まで進める | **AT 突入後のステージ動画(液晶の背景ムービー)が正常に再生される**(VM の Linux では黒画面になる既知事象があった。Windows の WebView2 では発生しない想定) |
| 2 | BGM ボタンを押す + 適当に遊技する | **BGM が実音で再生され、レバーオン・停止・払出・演出の SE が鳴る**(VM は音声デバイス無しで実音未確認だった)。ステージ切替時は BGM がクロスフェード |
| 3 | 起動直後 | ウィンドウ(760×1000)に筐体・リール窓・背景動画ループが表示される。動画が止まらない |
| 4 | レバーオン(Space)→ 停止ボタン(Z・X・C またはクリック) | Web 版と同じくリール回転 → タイミング目押しで停止(スベリ最大 4 コマ) |
| 5 | 予告・連続演出ムービー(前兆中〜連続演出 4G) | ムービーが再生される(仮素材。中身の絵は仮のままで正しい) |
| 6 | 通しフロー(`docs/STEP3_VERIFICATION.md` の 3-4 の手順) | 前兆 → 連続演出 → AT → エンディング → 通常復帰が途切れず進む |
| 7 | ウィンドウリサイズ(最小 520×720 まで) | レイアウトが崩れず追随する |

## 4. 確認できない(仕様上そのままで正しい)もの

- 演出ムービー・BGM/SE の中身は仮素材のまま(実素材差し替えは STEP 4f = 入稿待ち)
- exe のアイコンは Tauri 既定の仮アイコン(実アイコンは素材入稿後)
- コード署名なし(SmartScreen 警告が出るのは正常)
- 履歴グラフ・永続化・ペナルティ等は STEP 6

## 5. 問題を見つけたら

- 発生時の操作(どのチェック項目か・操作手順)、画面のスクリーンショット、
  可能なら Windows のバージョンを添えて次の AGENT へ依頼する。
- 特に #1(AT ステージ動画)・#2(実音)が NG の場合は「Windows でも発生する」という
  切り分け情報自体が重要(VM 限定の制約ではなくコード起因の可能性が高くなる)。
